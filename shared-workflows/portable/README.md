## Conventions

- Keep skill content portable and tool-neutral where possible
- Prefer Markdown instructions for reuse across Pi, Claude Code, and Codex
- Put tool-specific adapters in their own integration layer, not in the shared skill body
- Treat the shared repo as the canonical source of truth
