# Local markdown issue tracker

This repository tracks work in local markdown files under `.scratch/`.

Use this file only if the repo actually uses local markdown issue tracking.

## Notes
- Use `.scratch/<feature>/` for issue notes and slices.
- Create the `.scratch/` directory only if local issue tracking is chosen.
- Keep filenames stable and easy to sort.
- Repo docs still live under `docs/agents/`.
